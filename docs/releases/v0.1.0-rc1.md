# Dietary Exposure MCP v0.1.0-rc1

Status: public-review candidate; not a stable regulatory release

This candidate exposes the first reviewable Dietary Exposure MCP package for
food-mediated oral exposure screening and governed dietary evidence handoffs.
It is published to invite technical and scientific scrutiny, not to confer
submission readiness or regulator acceptance.

## Highlights

- 49 typed, read-only MCP tools with closed-world annotations
- deterministic acute and chronic dietary intake calculations
- bounded, survey-summary, bootstrap, and uncertainty-support workflows
- reference-value, legal-limit, method, occurrence, and consumption registries
- contaminant, metals, trade-risk, interoperability, and scientific follow-up
  review artifacts
- version-pinned schemas, examples, manifests, validation dossiers, and
  downstream handoffs

## Scientific Source Posture

- The 2,274-record bulk reference-value layer is explicitly identified as the
  superseded OpenFoodTox 2.0 version-6 snapshot, not current OpenFoodTox.
- Current OpenFoodTox 3.0 metadata is represented separately.
- Acetamiprid ADI and ARfD are corrected to `0.005 mg/kg bw/day` and linked to
  the EFSA 2024 statement and EU Regulation 2025/158.
- Multiple authorities, populations, assessment years, qualifiers, and unit
  bases remain visible instead of being silently flattened.

See the [scientific source review](../reviews/v0.1.0-scientific-source-review.md)
and [OpenFoodTox 3.0 migration ADR](../adr/0007-openfoodtox-3-migration.md).

## Security And Verification

The local release gate verifies lint, vendored-source integrity, scientific
invariants, the complete test suite, 455 generated validation cases,
reproducible wheel and source builds, Bandit, dependency auditing, secret
scanning, Twine metadata, clean-environment installation, an SBOM, and SHA-256
checksums.

Streamable HTTP binds to loopback by default, rejects unauthenticated startup,
enforces Host/Origin allowlists, and limits request bodies to 1 MiB. This RC is
intended for local stdio evaluation, not an unauthenticated public endpoint.

## Install From The GitHub Pre-Release

```bash
uv tool install ./dietary_mcp-0.1.0-py3-none-any.whl
dietary-mcp
```

Verify downloaded assets with the attached `SHA256SUMS` file before installing.

## Known Limitations

- External scientific signoff is not complete.
- The bulk OpenFoodTox 3.0 migration remains a separate acceptance-gated task.
- WHO-derived GEMS/Food seed-profile reuse terms require separate attention for
  commercial redistribution; see `THIRD_PARTY_NOTICES.md`.
- The service is not a final regulatory decision engine, PBPK execution engine,
  proprietary PRIMo/DEEM engine, or submission portal.
- The RC remains on the stable MCP v1 SDK line; MCP v2 beta work is isolated.
- The `0.1.0` distribution is not published to PyPI during the RC phase.

## Reporting

Use the repository's scientific-correction form for source, unit, population,
or interpretation concerns. Use GitHub private vulnerability reporting for
security issues and do not place sensitive details in public issues.

Third-party attribution and data-rights boundaries are documented in
`THIRD_PARTY_NOTICES.md`. Original authority outputs remain the scientific and
legal sources of record.

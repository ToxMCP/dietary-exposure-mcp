# Dietary MCP v0.1.0 Scientific Source Review

Review date: 2026-07-17

Status: internal review complete; external scientific signoff still required

## Scope

This review challenged the currency, traceability, and runtime presentation of the
highest-impact reference-value records in the v0.1.0 release candidate. It also
checked whether hosted transport controls and MCP annotations match actual
behavior. Passing this review does not make Dietary MCP a final regulatory
decision engine.

## Findings and resolutions

### F-01 High: superseded bulk snapshot labeled current

The 2,274-record bulk extract came from `ReferenceValues_KJ_2023.xlsx`, while
EFSA released OpenFoodTox 3.0 in April 2026. Bulk records now reference the
checksum-pinned `efsa.openfoodtox.2023_snapshot`, carry `superseded` status,
and produce an explicit currency warning. The current `efsa.openfoodtox` source
record now describes OpenFoodTox 3.0.

### F-02 High: acetamiprid values contradicted the linked 2024 source

The repository linked EFSA's 2024 statement but retained the older 0.025 ADI and
ARfD. Both records now use 0.005 mg/kg bw/day, match the checksum-verified
OpenFoodTox 3.0 workbook, preserve authority conflicts, and link the 2025 EU
Regulation as binding MRL risk-management context.

### F-03 Medium: scientific context was buried in notes

Bulk records now expose qualifier, assessment label, assessment year,
population, and OpenFoodTox output ID as structured fields. Lookup requests can
filter by those fields and by source ID. Multiple population, year, authority,
or unit contexts are surfaced without selecting a canonical value.

### F-04 Medium: hosted HTTP had no application request-body bound

Hosted HTTP now defaults to `127.0.0.1` and rejects bodies over 1 MiB before MCP
dispatch. Deployments can set `DIETARY_MCP_MAX_REQUEST_BYTES` deliberately.
Authentication remains delegated to a reviewed gateway and startup remains
fail-closed without the explicit gateway opt-in.

### F-05 Low: MCP annotations overstated side effects

All 49 current tools return in-memory transformations. Export tools build
payloads but do not write or publish them. Their annotations now consistently
report read-only, non-destructive, idempotent, and closed-world behavior.

## Remaining gates

- Perform independent toxicologist or dietary-risk-assessor review of the
  curated values, assumptions, and intended-use wording.
- Migrate the bulk layer to OpenFoodTox 3.0 under ADR 0007 with explicit
  equivalence and conflict tests.
- Review OAuth or gateway authentication, scopes, audit logging, and deployment
  limits before exposing Streamable HTTP beyond loopback.
- Keep release status `draft_ready` until those human decisions are recorded.

Review date: 2026-07-17

complete; see `openfoodtox-3-migration-plan.md` and
`openfoodtox-3-high-impact-review.json`. This review still records the original

Status: independent review not approved; remediation complete; revised signoff required

Owner attestation update: on 2026-07-22, Ivo Djidrovski, PhD accepted all 16
high-impact records for governed screening in a disclosed project

## Scope

This review challenged the currency, traceability, and runtime presentation of the
highest-impact reference-value records in the v0.1.0 release candidate. It also
checked whether hosted transport controls and MCP annotations match actual
behavior. Passing this review does not make Dietary MCP a final regulatory
decision engine.

## Findings and resolutions

### F-01 High: superseded bulk snapshot labeled current

The original 2,274-record bulk extract came from
`ReferenceValues_KJ_2023.xlsx`, while EFSA released OpenFoodTox 3.0 in April
2026. The runtime bulk layer now contains 2,417 checksum-pinned 3.0 records with
UUID provenance. The old source remains catalogued as superseded and is no
longer referenced by runtime bulk records.

### F-02 High: acetamiprid provenance overstated regulatory status

The repository previously blurred a scientific proposal with later regulatory
follow-up and represented the acute value with a daily unit. The curated ADI
now uses 0.005 mg/kg bw/day, while the ARfD uses 0.005 mg/kg bw. The raw
OpenFoodTox 3.0 unit remains visible only in source provenance, and the 2024 EFSA
statement is the introducing scientific source. Regulation (EU) 2025/158 is
linked only as a follow-up MRL risk-management measure.

### F-03 High: HumanHealth benchmark-dose evidence was omitted

The first high-impact pass searched only the toxicological-reference-value
sheet and incorrectly reported five benchmark records as absent. Extraction
now covers `END_STUDY_REC.HumanHealth` and preserves raw field paths, units,
qualifiers, UUIDs, dossiers, and literature links. It finds exact support for
both acrylamide BMDL10 values, the inorganic-arsenic BMDL05, and both lead
BMDL01 values.

### F-04 High: temporal values were flattened into apparent conflicts

Glyphosate ARfD is represented as 1.5 mg/kg bw for the current 2023 review,
with the 2015 value of 0.5 retained as superseded. Imidacloprid is intentionally
different: the 2013 recommendation of 0.06 mg/kg bw did not unconditionally
replace the 0.08 mg/kg bw value used in EFSA's 2019 MRL review. Both are current
assessment contexts in one explicit conflict group so callers must select the
applicable context.

### F-05 Medium: scientific context was buried in notes

Bulk records expose qualifier, assessment label, assessment year, population,
and source ID as structured fields. Their sidecar preserves toxicological
document UUIDs, source bounds, dossier links, persistent identifiers, critical
endpoints, and uncertainty fields. Multiple population, year, authority, or
unit contexts are surfaced without selecting a canonical value.

### F-06 Medium: hosted HTTP had no application request-body bound

Hosted HTTP now defaults to `127.0.0.1` and rejects bodies over 1 MiB before MCP
dispatch. Deployments can set `DIETARY_MCP_MAX_REQUEST_BYTES` deliberately.
Authentication remains delegated to a reviewed gateway and startup remains
fail-closed without the explicit gateway opt-in.

### F-07 Low: MCP annotations overstated side effects

All 49 current tools return in-memory transformations. Export tools build
payloads but do not write or publish them. Their annotations now consistently
report read-only, non-destructive, idempotent, and closed-world behavior.

### F-08 High: generated qualifier provenance referenced a nonexistent column

The independent reviewer found 564 lower-bound ARfD records claiming an
`Arfd.lowerQualifier` path that is absent from the source workbook. Those paths
are now null. The release verifier requires every non-null source, unit,
qualifier, and descriptor path to resolve to a header in the pinned workbook
inventory and directly rechecks the inventory when the local XLSX is present.

## Remaining gates

- Obtain positive independent toxicologist or dietary-risk-assessor signoff on
  the remediated version 1.2 high-impact report and its new canonical hash.
- Retain the owner's accept decisions for all 16 high-impact records as review
  history. The owner attestation and prior independent `not approved`
  disposition do not satisfy the independent gate.
- Review OAuth or gateway authentication, scopes, audit logging, and deployment
  limits before exposing Streamable HTTP beyond loopback.
- Keep the scientific-review dossier status `draft_ready` until those human
  decisions are recorded. This does not prevent publication of screening-only
  software.
